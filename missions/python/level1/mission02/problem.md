## 문항 2: 서버 접근 로그 분석기

### 시험 정보
- 과정: AI 올인원
- 단계: AI·SW 기초
- 난이도: 1
- 권장 시간: 15분
- Pass 기준: 총 100점 중 70점 이상

### 문제

CSV 형식의 서버 접근 로그를 분석하여 리포트를 생성하는 프로그램을 작성합니다. CSV 파싱, 데이터 집계, 정렬, 파일 I/O, argparse CLI 능력을 측정합니다.

`template/access_log_sample.csv`에 샘플 데이터가 제공됩니다.

#### CSV 형식

| 컬럼 | 설명 | 예시 |
|------|------|------|
| `timestamp` | 접근 시간 (ISO 8601) | `2025-03-15T09:00:00` |
| `ip` | 클라이언트 IP | `192.168.1.1` |
| `method` | HTTP 메서드 | `GET`, `POST` |
| `endpoint` | API 경로 | `/api/users` |
| `status_code` | HTTP 상태코드 | `200`, `404`, `500` |
| `response_time_ms` | 응답시간 (밀리초) | `45`, `5200` |

#### 핵심 요구사항

1. **CLI 인터페이스**: `argparse`를 사용하여 `--log` (입력 CSV 경로)와 `--output` (출력 리포트 경로) 인자를 받습니다.
   ```bash
   python log_analyzer.py --log <CSV파일경로> --output <리포트출력경로>
   ```

2. **IP별 접근 횟수 TOP 5**
   - IP 주소별 접근 횟수를 집계합니다
   - 접근 횟수가 많은 순서로 상위 5개를 출력합니다
   - **정렬 기준**: 접근 횟수 내림차순, 동점 시 IP 주소 내림차순

3. **HTTP 상태코드 그룹별 비율**
   - 상태코드를 그룹별(1xx, 2xx, 3xx, 4xx, 5xx)로 분류합니다
   - 전체 요청 대비 각 그룹의 비율(%)을 계산합니다
   - 데이터에 해당 그룹이 존재하는 경우에만 출력합니다
   - 소수점 1자리까지 표시합니다

4. **느린 엔드포인트 TOP 3**
   - 엔드포인트별 평균 응답시간을 계산합니다
   - 평균 응답시간이 긴 순서로 상위 3개를 출력합니다
   - 소수점 1자리까지 표시합니다

### 제약 사항
- `response_time_ms`는 정수 또는 소수일 수 있습니다 (예: `33.7`)
- IP가 비어 있는 행이 있을 수 있습니다 (IP 집계에서 제외하되, 다른 분석에는 포함)
- 상태코드는 1xx~5xx 범위를 모두 처리하세요
- 동점 정렬 기준을 정확히 구현하세요
- 외부 패키지 설치 금지 (표준 라이브러리만 사용)

### 출력 형식

```
=== IP Access Top 5 ===
192.168.1.1: 5
10.0.0.5: 4
172.16.0.10: 3
192.168.2.20: 1

=== Status Code Distribution ===
2xx: 69.2%
3xx: 7.7%
4xx: 7.7%
5xx: 15.4%

=== Slowest Endpoints Top 3 ===
/api/orders: 3436.7ms
/api/products: 70.3ms
/api/users: 47.2ms
```

> 위 수치는 샘플 CSV 기준입니다. 샘플 데이터에는 1xx 상태코드가 없으므로 1xx 행은 표시되지 않습니다. 실제 채점 시 다른 데이터로 검증됩니다.

### 제출 방식

`log_analyzer.py` 1개 파일을 제출합니다.
