## 문항 3: 프롬프트 관리 프로그램

### 시험 정보
- 과정: AI 올인원
- 단계: AI·SW 기초
- 난이도: 1
- 권장 시간: 15분
- Pass 기준: 총 100점 중 70점 이상

### 문제

PBL 과정에서 배운 Python 핵심 문법(변수, 조건문, 반복문, 함수, list+dict)을 **프롬프트 관리 프로그램**이라는 도메인에 적용하는 시험입니다.

배포된 `template/prompt_manager.py` 파일에서 `TODO`로 표시된 7개 함수를 구현하세요.

프로그램은 메뉴 기반 REPL로 동작하며, AI 프롬프트를 추가/조회/검색/즐겨찾기 관리할 수 있어야 합니다.

### 구현할 함수 (7개)

| 함수 | 기능 | 배점 |
|------|------|------|
| `add_prompt(prompts)` | 새 프롬프트 추가 (빈 제목 검증 포함) | 13점 |
| `list_prompts(prompts)` | 전체 프롬프트 목록 출력 | 7점 |
| `filter_by_category(prompts)` | 카테고리별 필터링 | 7점 |
| `search_prompts(prompts)` | 키워드 검색 (제목 **또는** 내용) | 13점 |
| `show_detail(prompts)` | 프롬프트 상세 보기 | 8점 |
| `manage_favorite(prompts)` | 즐겨찾기 토글 (추가 ↔ 해제) | 7점 |
| `show_favorites(prompts)` | 즐겨찾기 목록 출력 | 7점 |

### 초기 데이터

프로그램 시작 시 3개의 프롬프트가 미리 등록되어 있습니다 (템플릿에 포함):

1. **블로그 글 작성 도우미** — 카테고리: 텍스트 생성, 즐겨찾기: O
2. **제품 썸네일 생성** — 카테고리: 이미지 생성, 즐겨찾기: X
3. **IT 컨설턴트 페르소나** — 카테고리: 페르소나, 즐겨찾기: X

### 메뉴 구조 (템플릿에 제공)

```
=== 프롬프트 관리 프로그램 ===
1. 프롬프트 추가
2. 프롬프트 목록 보기
3. 카테고리별 조회
4. 프롬프트 검색
5. 프롬프트 상세 보기
6. 즐겨찾기 관리
7. 즐겨찾기 목록
0. 종료
```

### 세부 요구사항

#### 1. 프롬프트 추가 (`add_prompt`)
- 제목, 내용, 카테고리를 순서대로 입력받습니다
- **빈 제목을 입력하면 프롬프트를 추가하지 않고 안내 메시지를 출력합니다**
- 카테고리는 번호로 선택합니다 (CATEGORIES 상수 활용)

#### 2. 목록 보기 (`list_prompts`)
- `"번호. [카테고리] 제목"` 형식으로 출력합니다
- 즐겨찾기인 경우 제목 뒤에 `⭐`을 표시합니다

#### 3. 카테고리별 조회 (`filter_by_category`)
- 카테고리 목록을 출력하고 번호를 입력받습니다
- 선택한 카테고리에 해당하는 프롬프트만 출력합니다

#### 4. 검색 (`search_prompts`)
- 키워드를 입력받아 **제목 또는 내용**에 포함된 프롬프트를 출력합니다
- 대소문자를 구분하지 않으면 더 좋습니다

#### 5. 상세 보기 (`show_detail`)
- 번호를 입력받아 해당 프롬프트의 전체 정보(제목, 카테고리, 내용, 즐겨찾기)를 출력합니다

#### 6. 즐겨찾기 관리 (`manage_favorite`)
- 번호를 입력받아 즐겨찾기 상태를 **토글**합니다
- 즐겨찾기 O → X, 즐겨찾기 X → O

#### 7. 즐겨찾기 목록 (`show_favorites`)
- `favorite`가 `True`인 프롬프트만 출력합니다

### 입력 순서 (함수별)

| 함수 | input() 호출 순서 |
|------|-------------------|
| `add_prompt` | `input("제목: ")` → (빈이면 return) → `input("내용: ")` → 카테고리 출력 → `input("선택: ")` |
| `filter_by_category` | 카테고리 출력 → `input("선택: ")` |
| `search_prompts` | `input("검색어: ")` |
| `show_detail` | `input("번호 입력: ")` |
| `manage_favorite` | `input("번호 입력: ")` |

### 제약 사항
- 외부 패키지 사용 금지 (표준 라이브러리만 허용)
- 프롬프트 데이터는 `list` + `dict` 구조로 관리
- 기능별 함수 분리 (def 5개 이상)
- 잘못된 메뉴 입력 시 크래시 없이 안내 메시지 출력
